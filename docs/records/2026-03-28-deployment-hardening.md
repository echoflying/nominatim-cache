# 2026-03-28 部署加固记录

## 背景

当前版本虽然可以在本地运行，但直接上服务器仍有若干风险：

- Docker 镜像构建流程不正确
- 生产环境允许默认后台密码
- 没有独立健康检查接口
- `db:migrate` 指向缺失脚本
- 环境变量文件位置不清晰
- PM2 日志目录依赖系统绝对路径

## 本次修改

### 1. 运行时配置加固

- 支持同时识别 `server/.env` 与仓库根目录 `.env`
- 保持系统环境变量优先级最高
- 在 `NODE_ENV=production` 时强制要求非默认后台凭证

### 2. 健康检查能力

- 新增 `/health` 路由
- 健康检查仅验证服务进程自身状态，不调用外部 Nominatim

### 3. Docker 部署修复

- Dockerfile 改为多阶段构建
- builder 阶段负责安装完整依赖和构建 TypeScript
- runtime 阶段仅保留生产依赖和编译产物
- 前端静态文件一并打包进镜像
- Compose 健康检查切换到 `/health`

### 4. 脚本与运维修复

- 补回 `server/src/scripts/migrate.ts`
- `init-db` 改为基于统一配置读取数据库路径
- PM2 日志输出改为 `server/logs/`

### 5. 前端认证修复

- 移除浏览器内置默认 `admin/admin123`
- 改为首次访问管理接口时输入账号密码
- 401 后会清除缓存并提示重新输入

## 验证项

- 新增部署回归测试 3 个：
  - env 文件候选路径
  - 生产环境默认凭证拦截
  - `/health` 接口可用
- `npm test` 通过
- `npm run build` 通过
