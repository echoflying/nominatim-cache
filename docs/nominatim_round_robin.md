# 本文档定义NOMINATIM数据源访问调度算法

基于Sequential Round Robin顺序轮询调度，结合断路器模式与指数退避封禁策略，对 A/B/C 三源进行单机本地状态管理，自动跳过封禁源、失败切换、到期恢复，超最大遍历次数则快速失败返回错误，并全量记录调用统计日志。



# 三源（A/B/C）轮询调度+智能封禁算法

## 基础定义

- 数据源：固定序列 `A → B → C`

- 运行环境：**单机单实例**，维护本地全局状态

- 核心模式：顺序轮询，单源调用失败自动切换下一个源



---



## 一、全局状态维护（单机本地存储）

为每个数据源独立维护4项核心状态，全局唯一：

1. **轮询指针**：自增数字，用于顺序选取数据源

2. **连续失败计数**：记录源连续调用失败的次数

3. **封禁状态标记**：布尔值，标记源是否被临时封禁

4. **封禁等级**：数字，用于计算指数递增的封禁时长（初始为0）

5. **封禁截止时间戳**：记录当前封禁周期的结束时间



---



## 二、失败与封禁判定规则

### 1. 失败判定范围（全部纳入计数）

满足以下任一条件，标记本次调用**失败**，计入连续失败计数：

1. 网络异常：请求超时、连接拒绝、域名解析失败

2. HTTP状态码：400、429、5xx系列错误

3. 特殊响应：HTTP 200 成功，但响应HTML内容包含`BLOCK`关键字

4. 所有非成功的调用场景（无地址结果也统一按失败统计）



### 2. 封禁触发规则

- 连续失败次数达到**阈值3次**，触发封禁

- 封禁时长采用**指数递增规则**，起始时长1分钟：

&nbsp; 封禁等级0 → 1分钟；等级1 → 2分钟；等级2 → 4分钟；等级3 → 8分钟……以此类推

- 触发封禁时：更新封禁截止时间戳、标记封禁状态、封禁等级+1



### 3. 成功重置规则

源调用成功后，**立即重置**：连续失败计数=0、封禁状态=未封禁、封禁等级=0、封禁截止时间戳清空



---



## 三、数据源可用性校验规则

1. 选取源前优先校验封禁状态与时间

2. 若当前时间 ≥ 封禁截止时间戳：**自动清除封禁**，重置所有异常状态，恢复为可用源

3. 若源处于封禁期且未到期：**直接跳过**该源，选取下一个轮询源



---



## 四、核心轮询调度执行流程

1. **初始化遍历计数**：设置最大遍历次数为`3次`，防止死循环，当前遍历次数=0

2. **计算候选源**：通过 `轮询指针 % 3` 确定当前候选数据源，指针自增1

3. **可用性校验**：检查候选源是否可用，封禁未到期则跳过，执行步骤2选取下一个源

4. **执行调用**：对可用源发起地理编码请求

5. **结果与状态处理**

&nbsp;  - 调用成功：重置该源所有异常状态，返回有效结果，流程结束

&nbsp;  - 调用失败：该源连续失败计数+1；达到3次则触发指数封禁，更新封禁状态与时长

6. **遍历次数控制**：当前遍历次数+1，达到最大3次则终止流程

7. **兜底处理**：遍历3次后仍无可用源/调用失败，**直接返回全局错误**，流程结束



---



## 五、统计日志规则

1. 记录维度：每个源的总调用次数、成功次数、失败次数、封禁次数、当前封禁等级

2. 记录时机：每次调用完成/状态变更时同步更新统计数据

3. 约束条件：统计仅做日志留存，**不干预任何调度、封禁、恢复逻辑**



---



## 六、边界约束汇总

1. 轮询序列：严格遵循`A→B→C`循环顺序，无优先级偏向

2. 最大重试：单请求最多遍历所有源3次，强制终止避免死循环

3. 全源封禁：直接返回错误，不强制调用任何源

4. 封禁恢复：到期自动解除，无人工干预

5. 全局配置：A/B/C三个源使用统一的失败阈值、封禁规则，无差异化配置
